# 수동 검증 기록

자동 검증은 [verification-report.md](verification-report.md)가 자동으로 남긴다.
**여기는 사람이 직접 본 것**을 적는다 — 안 그러면 다음에 "했나 안 했나"를 모른다.

절차와 각 항목이 무엇을 보는지는 [verification-spec.md](verification-spec.md)의 "수동 검증".

기록 규칙:
- 확인한 **커밋**과 **날짜**, **누가** 봤는지 적는다
- **못 본 것은 "미검증"이라고 적는다.** 통과와 미검증을 섞지 않는다
- 코드가 바뀌면 예전 기록은 그 커밋에 대한 것일 뿐이다 — 배포 전에 다시 본다

### M-5에서 잡은 것 (자동 검증 9개가 전부 통과했는데도 있던 버그)

영어로 바꿨더니 **UI는 영어인데 옵션 원문은 한국어**인 반쪽 화면이 떴다.
동적 import에 붙인 `with { type: "json" }`이 **Vite dev에서만** 로케일 로딩을 깨뜨렸다
(빌드본은 JSON을 번들에 넣어 멀쩡 → 자동 검증도 프로덕션도 통과).

→ `preloadLang`으로 고쳤다: 언어를 바꾸기 **전에** 게임 데이터를 받아 둔다. 실패하면 아무것도
안 바뀌므로 반쪽 화면이 구조적으로 못 생긴다. 실패는 `console.error`로 시끄럽게 남는다.

**교훈**: dev/prod가 갈리는 버그는 자동 검증이 원리적으로 못 잡는다. 언어 전환은 반드시 눈으로 본다.
