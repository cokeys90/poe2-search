# 즐겨찾기 플로팅 창 전환 — 설계·구현 계획서

작성일: 2026-07-13 · 상태: **설계안(미구현)** · 대상: `poe2-search`

---

## 1. 배경 / 문제

현재 즐겨찾기는 `RightPanel.jsx` — 우측에 고정된 폭 320px(`w-80`) 사이드바다.

- **좁다.** 카드 한 장에 이름 + 패턴이 들어가는데 폭 320px에서 둘 다 말줄임된다.
- **크기를 못 바꾼다.** 사용자가 넓히거나 좁힐 방법이 없다.
- **`xl` 미만에서 아예 안 보인다** (`hidden xl:flex`). 노트북 창 폭이 작으면 즐겨찾기 자체를 못 쓴다.
- 레이아웃을 밀어내므로 열면 옵션 목록이 좁아진다. `ScrollFab`도 패널 유무에 따라 위치를 보정(`rightInset`)해야 한다.

## 2. 목표 / 비목표

**목표**

1. 즐겨찾기를 **앱 화면 위에 떠 있는 드래그·리사이즈 가능한 플로팅 창**으로 교체한다.
2. 창의 **위치·크기·표시모드**를 localStorage에 저장하고, 다시 열면 그 상태 그대로 복원한다.
3. **리스트형 / 카드형** 두 가지 표시모드를 토글 버튼으로 전환한다. (리스트형 = 한 줄, 카드형 = 현재 모양)
4. 레이아웃을 밀지 않으므로 옵션 목록 폭이 창 열기와 무관해진다.

**비목표 (이번 범위 아님)**

- 실제 브라우저 팝업 창(`window.open`) — 검토했으나 팝업 차단·스타일 주입·창 간 상태 동기화 비용이 이득보다 큼. 앱 내부 플로팅 창으로 확정.
- 여러 창 동시 띄우기, 창 도킹/스냅, 즐겨찾기 검색/정렬 기능.
- 즐겨찾기 데이터 스키마(`poe2-search:favorites` v2) 변경 — **건드리지 않는다.**

## 3. UX 사양

### 3-1. 창 골격

```
┌─ ⭐ 즐겨찾기 ──────────── [☰|▦] [＋그룹] [✕] ┐   ← 헤더: 드래그 핸들
│ ▾ 미분류                                  2 │
│   ⭐ 균질 T15 파밍                           │
│   ⭐ 서판 균열 3옵션                          │
│   ＋ 여기에 현재 검색 추가                     │
│ ▾ 서판                                    1 │
│   ...                                       │
│                                          ◢  │   ← 우하단 리사이즈 핸들
└─────────────────────────────────────────────┘
```

- **모달 아님(non-modal).** 스크림 없음, 바깥 클릭해도 안 닫힘. 창을 띄운 채로 옵션을 고르고 → 창의 `+ 추가`를 눌러 저장하는 게 핵심 동선이라 여기서 모달이면 기능이 죽는다. (`Modal.jsx`는 재사용하지 않는다.)
- 닫기: 헤더 `✕` 버튼, `Esc` 키.
- 열기: 지금과 동일하게 헤더의 ⭐ 버튼 토글. 단 **`xl` 조건을 제거**해 어느 폭에서든 열 수 있게 한다.
- z-index: 다이얼로그(`z-50`)보다 아래. 창 = `z-40`. 즐겨찾기 불러오기/그룹 삭제 확인 다이얼로그가 창 위에 떠야 하기 때문.

### 3-2. 드래그

- **헤더 영역만** 드래그 핸들. 헤더 안의 버튼(뷰 토글·새 그룹·닫기)은 드래그에서 제외.
- `pointerdown` → `setPointerCapture` → `pointermove`로 x/y 갱신 → `pointerup`에서 저장.
- 창이 화면 밖으로 완전히 나가지 않도록 클램프: 헤더가 최소 `48px`는 뷰포트 안에 남게.

### 3-3. 리사이즈

- 우하단 코너 핸들 하나 (`◢`). 상하좌우 8방향은 과함 — 이번엔 안 한다.
- 제약: `minW 260 / minH 240`, `maxW·maxH = 뷰포트 크기 - 여백`.
- 드래그와 동일하게 pointer 이벤트, `pointerup`에서 저장.

### 3-4. 기본 크기·위치 (⚠️ 확정 필요 — 6번 참조)

- 사용자 요청: "보통 핸드폰 해상도의 0.7배". 요즘 폰 CSS 논리 해상도 ≈ `390×844` → **0.7배 = 273×591**.
- 273px는 현재 카드형(아이콘 + 이름 + hover 액션 3개)이 들어가기엔 빠듯하다. → **기본값 `320×600`을 제안**한다 (폰 세로비 유지, 카드형이 안 깨지는 최소선).
- 기본 위치: 뷰포트 **우하단에서 여백 24px**. (헤더의 ⭐ 버튼 근처가 아니라 우하단인 이유: 상단 결과 바를 가리지 않기 위해.)

### 3-5. 표시모드 — 리스트형 / 카드형

헤더 우측 세그먼트 토글 2버튼 (기존 `Segmented.jsx` 재사용 가능, 아이콘만이면 자체 구현).

| | 카드형 (현재) | 리스트형 (신규) |
|---|---|---|
| 높이 | 2줄 (이름 + 패턴) | 1줄 |
| 내용 | 아이콘 · 이름 · 패턴(모노) | 아이콘 · 이름 · (여백 있으면 패턴 회색) |
| 액션 | hover 시 우측 3버튼(덮어쓰기·이름변경·삭제) | 동일하되 hover 시 이름을 가리므로 패턴 숨김 |
| 여백 | `py-2` + 카드 테두리 | `py-1`, 테두리 없이 hover 배경만 |
| 용도 | 패턴까지 확인하며 고를 때 | 항목 많을 때 한눈에 |

- 두 모드 모두 **드래그 앤 드롭(그룹 간 이동·정렬)은 그대로 동작**해야 한다.
- 리스트형에서 좁은 창(<300px)이면 패턴은 자동 숨김.

### 3-6. 좁은 화면 fallback

`isWide`(현재 `useMediaQuery`로 판단)가 false면 — 즉 모바일·좁은 창이면 — 플로팅 창 대신 **전체화면 시트**로 렌더한다. 위치/크기 저장은 무시하고 `inset-0`. (좁은 화면에서 드래그 가능한 창은 쓸모가 없다.)

### 3-7. 뷰포트 리사이즈 대응

브라우저 창이 작아져 저장된 좌표가 화면 밖이 되는 경우 → `resize` 이벤트에서 클램프해 다시 안으로 끌어들인다. (저장은 하지 않고 표시만 보정하면 브라우저를 다시 키웠을 때 원위치 복원 — 이쪽이 덜 놀랍다.)

## 4. 설계

### 4-1. 파일 구성

```
src/
  components/
    FloatingWindow.jsx     [신규] 범용 창 셸 — 드래그·리사이즈·클램프·좁은화면 fallback
    FavoritesWindow.jsx    [신규] 즐겨찾기 내용 — 헤더 액션 + 뷰 토글 + 그룹/항목 렌더
    FavItem.jsx            [신규] 카드형·리스트형 한 항목 (variant prop)
    RightPanel.jsx         [삭제] 내용은 FavoritesWindow / FavItem으로 이관
  hooks/
    useWindowState.js      [신규] {x,y,w,h,view} 상태 + localStorage 영속 + 클램프
  lib/
    storage.js             [수정] loadWinState / saveWinState 추가
  App.jsx                  [수정] RightPanel → FavoritesWindow, 레이아웃 정리
```

**책임 분리 원칙**: `FloatingWindow`는 즐겨찾기를 모른다(children만 받는 순수 창). `FavoritesWindow`는 드래그·리사이즈를 모른다. 나중에 다른 패널을 창으로 띄울 때 재사용 가능하되, **지금 필요 없는 옵션(도킹·최소화·다중창)은 만들지 않는다.**

### 4-2. 저장 스키마 (신규 키)

즐겨찾기 데이터(`poe2-search:favorites`)와 **분리**한다. 창 상태는 UI 설정이지 사용자 데이터가 아니고, 스키마 마이그레이션 리스크를 섞지 않기 위함. `poe2-search:pins`와 같은 층위.

```js
// key: "poe2-search:favwin"
{
  version: 1,
  x: 1120, y: 300,     // 뷰포트 좌상단 기준 px
  w: 320,  h: 600,
  view: "card",        // "card" | "list"
  open: false          // 재접속 시 창 자동 복원 여부
}
```

- 파싱 실패·버전 불일치 → 기본값. (`loadPins`와 동일한 방어 패턴)
- `open`을 저장할지는 취향 문제 → **저장한다.** "그 상태 그대로 보이게"라는 요청에 창이 열려 있었는지도 포함된다고 본다.
- 그룹 접힘 상태(`collapsed`)는 현재 컴포넌트 로컬 state라 새로고침 시 날아간다. **이번 범위에서 제외** — 필요하면 후속으로 `favwin`에 `collapsedGroups: []` 추가.

### 4-3. 드래그/리사이즈 구현 노트

- 위치는 **`left`/`top` 인라인 스타일**로. (`transform`은 내부 HTML5 DnD의 drop 좌표 계산과 섞이면 헷갈린다.)
- pointer 이벤트 중에는 `useState`가 아닌 **`useRef` + 직접 DOM 스타일 갱신**으로 리렌더를 피하고, `pointerup`에서 한 번 setState + 저장한다. (창 안에 즐겨찾기 트리가 통째로 있어 매 프레임 리렌더는 눈에 띄게 버벅인다.)
- 드래그 중 `user-select: none`, `cursor: grabbing`.
- **HTML5 DnD 충돌 주의**: 항목 카드는 `draggable`이라 `pointerdown`이 먹힌다. 창 드래그는 헤더에만 붙이므로 충돌 없음 — 다만 헤더에 `draggable` 속성을 절대 붙이지 말 것.

### 4-4. App.jsx 변경

- `rightOpen` → `favWinOpen` (의미 변경: 사이드바 열림 → 창 열림). 초기값은 저장된 `open`.
- 헤더 ⭐ 버튼: `isWide` 조건 제거(항상 노출).
- `<div className="flex min-h-0 flex-1">` 안에서 `<RightPanel/>` 제거 → `<main>`만 남으므로 래퍼 정리.
- `ScrollFab`의 `rightInset={rightPanelOpen ? 312 : 24}` → **`24` 고정**. (창은 레이아웃을 밀지 않는다. FAB와 창이 겹칠 수는 있으나 창을 옮기면 되고, FAB는 `z-30`으로 창 아래에 두면 자연스럽다.)
- `FavoritesWindow`는 레이아웃 밖(다이얼로그들과 같은 층)에 렌더.

기존 `useFavorites` 훅, `snapshot()`, `applyFavorite()`, `requestLoadFavorite()`, `ConfirmDialog` 연결은 **그대로 재사용** — props 전달 대상만 바뀐다.

## 5. 구현 순서 & 검증

| # | 단계 | 검증 |
|---|------|------|
| 1 | `storage.js`에 `loadWinState`/`saveWinState` + `useWindowState` 훅 | 콘솔에서 값 넣고 새로고침 → 복원 확인 |
| 2 | `FloatingWindow.jsx` (드래그·리사이즈·클램프) + 더미 children | 창 이동·리사이즈, 화면 밖으로 안 나감, 새로고침 후 위치/크기 유지 |
| 3 | `FavItem.jsx` — 카드형(현 동작 이식) + 리스트형 | 두 모드에서 로드/덮어쓰기/이름변경/삭제 모두 동작 |
| 4 | `FavoritesWindow.jsx` — 헤더(뷰 토글·새 그룹·닫기) + 그룹 섹션 + DnD | 그룹 간 이동·정렬이 두 모드 모두에서 동작 |
| 5 | `App.jsx` 배선, `RightPanel.jsx` 삭제, `ScrollFab` inset 원복 | 옵션 목록 폭이 창 열기와 무관, FAB 정상 |
| 6 | 좁은 화면 fallback | 창 폭 줄여 `xl` 미만 → 전체화면 시트로 전환 |

**최종 검증 (dev 서버 + 브라우저 — `verification.md` 3항)**

- 골든패스: 옵션 선택 → 창 열기 → 그룹 `+추가` → 이름 변경 → 창 이동/리사이즈 → 새로고침 → 창이 같은 위치·크기·모드로 뜨고 항목 로드 성공.
- 엣지: 저장된 좌표가 화면 밖(브라우저 축소 후 새로고침) / localStorage 값 손상 / 즐겨찾기 0개 / 그룹 삭제 확인 다이얼로그가 창 위에 뜨는지 / `Esc` 닫기.
- 회귀: 핀·가격·타락 필터, 탭 전환, `ScrollFab`.
- `npm run build` 통과.

## 6. 확정 필요 (구현 전 답 필요)

1. **기본 크기** — 요청대로 `273×591`(폰 0.7배 그대로)로 갈지, 카드형이 안 깨지는 `320×600`으로 갈지. → **`320×600` 권장.**
2. **기본 위치** — 우하단(제안) vs 화면 중앙 vs 기존 사이드바 자리(우측 상단).
3. **`open` 상태 저장** — 새로고침 시 창을 자동으로 다시 띄울지. → **띄우는 쪽 권장.**
4. **리스트형에서 패턴 표시** — 이름 옆에 회색으로 같이 보여줄지, 이름만 보여줄지. → **여백 있으면 표시 권장.**

## 7. 리스크

- **DnD × 창 드래그 충돌**: 헤더/본문의 포인터 이벤트 경계를 잘못 잡으면 항목을 끌 때 창이 따라 움직인다. → 창 드래그는 헤더 전용, 본문에는 pointer 핸들러를 붙이지 않는다.
- **리렌더 성능**: 드래그 중 setState 하면 즐겨찾기 트리 전체가 매 프레임 리렌더. → 4-3의 ref 방식으로 회피.
- **`RightPanel.jsx` 삭제**: 다른 참조가 없는지 확인 후 삭제(현재 `App.jsx` 한 곳만 import).
