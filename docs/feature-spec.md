# 기능 명세

이 앱이 무엇을 하는가. 기능마다 **ID**를 붙인다 — 검증 명세([verification-spec.md](verification-spec.md))와
검증 보고서([verification-report.md](verification-report.md))가 이 ID로 기능을 가리킨다.

기능을 더하거나 고치면 **여기를 먼저 고치고**, 그 기능을 어떻게 검증할지 검증 명세에 적은 뒤 구현한다.
검증 수단이 없는 기능은 "수동"이라고 정직하게 적는다. 검증 안 한 것을 통과했다고 적지 않는다.

> 도메인 규칙(경로석 3영역, 복합 모드, 조각 생성 규칙, 수치식)은 [`../CLAUDE.md`](../CLAUDE.md)에 있다.
> 여기는 **무엇을 하는가**만 적는다. **왜 그렇게 하는가**는 CLAUDE.md다.

## 핵심 — 이 앱의 존재 이유

### F-01 옵션 데이터

경로석·서판의 옵션 목록을 언어별로 갖고 있다. 총 **126개**.

- 경로석: 옵션(상단 6) · 접두어(15) · 접미어(20)
- 서판: 종류별 고정 옵션(8) · 공통 접두어(13) · 공통 접미어(11) · 종류별 고유 접미어(53)
- 서판 종류 8가지: 방사능 · 균열 · 탐험 · 환영 · 의식 · 감독관 · 심연 · 사원
- 옵션마다: 원문(`text`) · 검색조각(`frag`) · 겹침수(`overlap`) · 부가 옵션(`extra`) · 거래소 스탯(`stat_id`)
- 옵션의 정체성은 **언어무관 안정키**(`ws.pre.deal_extra_fire`) — 텍스트가 아니다

**출처**: 원문·조각은 poe2db, 경로석 상단 6옵션은 인게임 캡처. **번역하지 않는다** (CLAUDE.md §6-0)

### F-02 옵션 선택

옵션을 골라 검색 조건을 만든다.

- 옵션 목록 클릭 순환: 없음 → 포함 → 제외 → 제거
- 고른 조건은 결과 바에서 **필수 / 선택 / 제외** 세 그룹으로 나뉜다 (옵션마다 따로)
  - **필수(AND)**: 반드시 있어야 · **선택(OR)**: 이 중 하나라도 · **제외(NOT)**: 없어야
  - 그룹 이동은 두 가지 — 칩의 **별(☆) 토글**(선택↔필수 한 번에) · **드래그**로 세 존 사이 이동
  - 새로 고른 옵션은 기본 **선택**이다 (필수는 명시적으로 올린다)
- 수치 입력 — "N 이상"만 지원 (거래소·인게임 모두 min 기반)
- 옵션 찾기(부분 문자열)
- 서판은 종류를 먼저 고른다 — 종류가 고유 접미어 목록을 결정한다

### F-03 검색어 생성 🔴 이 앱의 심장

고른 조건을 **인게임 검색창에 붙여넣을 정규식**으로 만든다.

- 조각 + 수치식 결합 — 수치는 원문에서 값이 조각보다 앞이면 앞에 붙인다
- 값 뒤 앵커 `[%(]` — 게임의 "어드밴스드 모드 설명" 켬/끔 양쪽에서 잡히게
- 범위 압축 — "33 이상"을 옵션 범위 안에서만 표현 (`(3[3-9]|[4-9].|\d..)%`)
- 등급(`\b15등급`) · 타락(`"타락"` / `"!타락"`) · 가격(`\b3 exalted`)
- 결합은 **옵션별**(F-02) — 필수는 각자 세트(공백=AND), 선택은 한 세트로 `|`(OR), 제외는 `!`.
  예: `"^몬.*율" "견.*경|성.가" "!내.무"` (필수1 · 선택2 OR · 제외1)
- **250자 제한** — 게임 검색창의 상한

### F-04 서판 고정 옵션 (잔여 사용 횟수)

서판마다 늘 붙어 있는 옵션(`지도에 {종류} 추가 / 잔여 사용 횟수 (1—10)회`).
안 쓴 서판(10회)만 찾는 게 거래의 기본이라 **기본 선택**이며, 끌 수 있다. 종류를 바꾸면 따라 바뀐다.

### F-05 필터

- 경로석 등급 (1~15)
- 타락 (상관없음 / 타락만 / 비타락만)
- 가격 (정확히 / 범위, 화폐 3종) — 화폐명은 언어와 무관하게 영어

## 거래소 연동

### F-06 거래소로 내보내기

고른 조건을 공식 거래소 URL(`?q=<JSON>`)로 만들어 새 탭에서 연다. **API 호출·인증 없음.**

- 옵션 → `stats`: 필수→`and` · 선택→`count`(min:1) · 제외→`not` · 경로석 상단 6옵션 → 엔드게임 필터(`map_iir` 등)
- 등급 → `map_tier` · 가격 → `trade_filters` · 타락 → `misc_filters`
- **`query.type`(기본 타입명)** — 안 넣으면 서판 8종이 섞인다. **그 거래소의 언어**로 넣는다
- 거래소 3곳: 글로벌(언어별 서브도메인) · 카카오(한국) · 대만
- 거래소로 못 보내는 옵션은 **조용히 빠지지 않고** 사용자에게 이름으로 보고된다

### F-07 거래소에서 가져오기 (북마클릿)

거래소 페이지에서 북마클릿을 누르면 그 검색 조건이 이 앱으로 넘어온다.

- 거래소 페이지의 `window.tradeOpts.state`를 읽어 `#trade=…`로 앱을 연다 — **거래소 서버에 요청하지 않는다**
- 거래소 그룹을 그대로 복원한다: `and`→필수 · `count`→선택 · `not`→제외 (F-02의 왕복 보존)
- 짧은 검색 ID(`/rPBBYV9GuQ`)로 되돌리는 건 하지 않는다 (레이트리밋·CORS)
- 앱이 모르는 조건은 스탯 이름으로 배너에 보고
  - 그 이름표는 거래소 API에 있는데 CORS가 없어 브라우저가 못 읽는다 → **Cloudflare 워커**(`worker/`)가
    대신 불러 붙여준다. ⚠️ 이름표도 **거래소마다 언어가 다르다** → 어느 거래소를 읽을지 넘긴다
    (`/stats?origin=…`). 워커는 거래소만 허용한다 — 아무 URL이나 대신 불러주면 오픈 프록시가 된다

## 사용자 편의

### F-08 즐겨찾기

검색 조건을 이름 붙여 저장한다. 플로틱 창(non-modal — 띄운 채로 옵션을 고를 수 있다).

- 그룹 만들기·이름 바꾸기·지우기 / 항목 저장·덮어쓰기·이름 바꾸기·지우기
- 그룹 간 **드래그로 이동**
- 리스트/카드 두 가지 보기
- 항목에서 바로 거래소 열기
- 저장 키: `poe2-search:favorites`

### F-09 핀

지금 값을 "다음에도 유지"로 고정한다 (가격·타락·등급·옵션). 저장 키: `poe2-search:pins`

### F-10 옵션 개인화

그룹 안에서 옵션을 **드래그로 정렬**하고 **숨길** 수 있다. 저장 키: `poe2-search:optprefs`

### F-11 설정

- **언어** (10개) — 게임 클라이언트와 맞춰야 인게임 검색이 통한다
- **거래소 서버** (자동 / 글로벌 / 카카오 / 대만) — 언어와 별개 축
- **리그** — 거래소마다 다르다 (대만은 리그 id가 중국어)
- 즐겨찾기 창 초기화 / 옵션 순서·숨김 초기화

## 다국어 · 배포

### F-12 다국어 (10개 언어)

`kr us jp tw ru pt th fr de sp`

- **게임 데이터** (옵션 원문·조각·토큰·타입명): 언어별 로케일 파일. poe2db에서 온다
- **화면 문구**: `src/i18n/{lang}.json` (우리가 쓴다 — 게임 용어는 poe2db와 대조)
- **폰트**: Pretendard(라틴·한글·가나) + 필요한 언어만 Noto를 지연 로딩(키릴·태국어·한자)
- 언어를 바꿔도 새로고침하지 않는다 — 고른 옵션이 날아가면 안 된다
- ⚠️ **검색어 생성은 로케일 데이터만 쓴다.** 화면 문구(손번역)는 구조적으로 들어갈 수 없다

### F-13 라우팅 · SEO

주소는 `/{언어}/{페이지}/` — 둘 다 생략할 수 있다. **10개 언어 × 3페이지 = 30개 URL.**

- 언어: `/` (한국어, x-default) · `/en/` `/ja/` `/zh-tw/` `/ru/` `/pt-br/` `/th/` `/fr/` `/de/` `/es/`
- 페이지: 없음(허브) · `/waystone/` · `/tablet/` — 세그먼트는 **영어 고정**(앱의 `tab`과 같은 낱말)
- 빌드가 **조합마다 진짜 HTML 파일**을 찍어낸다 (크롤러는 JS 실행 전 `<head>`를 본다)
- 각 파일: 자기 언어·페이지의 title·description·og·JSON-LD + **같은 페이지의** 다른 언어를 가리키는
  hreflang (`/waystone/`의 짝은 `/en/waystone/`이지 `/en/`이 아니다)
- 본문도 페이지마다 다르다 — 경로석 페이지엔 경로석 옵션 41개, 서판 페이지엔 서판 옵션 85개.
  같은 내용을 URL만 바꿔 내면 중복으로 묶여 나누는 뜻이 없어진다
- 사이트맵 자동 생성 (30개 URL, 각자 hreflang 묶음)
- **주소가 저장값을 이긴다** — `/en/` 링크를 받으면 영어를 본다
- **주소가 탭을 정한다** — `/waystone/`로 들어오면 경로석이 열린다. 탭을 고르면 주소가 따라온다
  (탭 버튼·즐겨찾기·거래소 가져오기 전부). 허브는 "아직 안 골랐음"이라 첫 화면에서 주소를 덮지 않는다
- ⚠️ SEO 문구에 **게임 용어를 손으로 적지 않는다** — `seo.json`은 `{waystone}` `{tablet}` `{types}`만
  두고 빌드가 로케일에서 채운다(`scripts/lib/item-words.mjs`). §6-0의 경계가 SEO에도 그대로 선다

### F-14 저장 · 이관

localStorage에 영속. 언어·거래소·리그·테마·즐겨찾기·핀·옵션순서·창 위치.

- ⚠️ 다국어 도입 전엔 옵션 id가 **한국어 텍스트의 해시**였다 → 안정키로 1회 이관(`legacyIdMap.js`)
- 이관이 실패하면 기존 사용자의 즐겨찾기·핀이 통째로 사라진다

### F-15 화면 셸

테마(다크/라이트) · 반응형(좁은 화면은 전체화면 시트) · 플로팅 창 이동·크기조절 · 스크롤 점프

## 지원하지 않는 것 (의도적)

- **간체 중국어(cn)** — 게임이 중국 전용 클라이언트라 경로석 상단 6옵션·타락·등급을 캡처할 수 없다
- **거래소 max 조건** — 내보내기가 min만 지원한다 → 음수·감소 계열 옵션 일부는 거래소로 못 보낸다
- **검색 ID로 가져오기** — 레이트리밋·CORS로 구조적으로 불안정 (북마클릿만 지원)
