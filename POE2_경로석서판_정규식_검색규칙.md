# PoE2 경로석·서판 정규식 검색 규칙

> 출처: poe2db.tw/kr (옵션 원문) · poe.kakaogames trade2 API (거래소명) · 엔진 참고 poe2.re (veiset/poe2.re)
> 검증: 0.5.x (Runes of Aldur) · 작성 코크YS
> 최종 갱신: 경로석 3분할(옵션/접두/접미) · poe2.re식 범위 수치식 · 방사능 서판 포함

경로석·서판 옵션 검색용 정규식 조각 생성 규칙. 세션이 초기화돼도 이 규칙대로 하면 일관된 조각이 나온다.

---

## 0. 문법

게임 내 검색은 **부분 정규식**을 지원한다.

- `.` = 아무 글자 1개 / `.*` = 0글자 이상
- `|` = OR / `!` = 제외(조각 맨 앞) / `^` = 줄 시작 / `\s` = 공백
- `" "` = 검색 세트 감싸기. 세트끼리는 공백으로 구분: `"포함" "!제외"`
- 최대 250자

거래소(공식 trade) 검색은 별개 — 정식 스탯명으로 찾는다(시트 `거래소명(#)` 참고).

---

## 1. 경로석 구조 (중요)

경로석 = **옵션 + 접두어 + 접미어** 3영역.

- **옵션** (6종): 옵션 목록 상단 영역. 부활 횟수 / 아이템 희귀도 / 무리 규모 / 몬스터 희귀도 / 경로석 출현 확률 / 몬스터 효율.
  - 실제로 경로석은 이 상단을 주로 검색한다.
  - poe2db 원문에 값 범위가 없어(경로석 등급/합산 표시) 수치는 '입력값~999'로 검색. 부활 횟수만 0~6.
- **접두어 / 접미어**: poe2db 각 모디파이어의 **첫 줄만** 실제 옵션. 딸린 부가효과 줄은 제외.
  - 티어별로 값만 다른 건 하나로 병합.

## 서판 구조

서판 = **공통 접두(13) + 공통 접미(9) + 종류별 고유 접미**.
- 종류: 균열7 · 탐험8 · 환영9 · 의식9 · 감독관8 · 심연8 · 사원3 · **방사능0(공통만)**.
- 방사능(Irradiated)은 전용 고유 옵션이 없어 공통 옵션으로 검색.

---

## 2. 검색조각 생성 규칙

**그 옵션만 찾는 최소 정규식**.

1. **수치는 넣지 않는다.** 값 무시, 옵션 종류만 특정.
2. **고유성 우선.** 같은 아이템(경로석/서판) 안에서 그 옵션만 매칭.
3. **우선순위:**
   - (1) 한 단어로 고유하면 그 단어 → `화염`, `정확도`, `골드`
   - (2) 안 되면 핵심명사 첫 글자 쌍 → `아.희`
   - (3) 겹치면 고유해지는 다른 글자 쌍 → `생.*최` vs `생.*회`
4. **위치어·공통어미 제외.** 위치어=지도/지역/몬스터, 공통어미=증가/증폭/등장/확률.
5. **경로석·서판 별개** → 교차 중첩 무시. 같은 풀 내 중첩은 수치로 최종 특정.
6. `.` vs `.*`: 사이 1글자면 `.`, 여러 글자면 `.*`.

### 고유 글자쌍 예시

| 옵션 | 조각 |
|---|---|
| 최대 **생**명력의 …에너지 보호막 **최**대치로 획득 | `생.*최` |
| 플레이어의 **생**명력 및 …**회**복 속도 감폭 | `생.*회` |

---

## 3. 수치식: "N% 이상" (poe2.re 방식) ⭐

옵션 범위(min~max)를 알므로 **그 범위 안에서만** 표현 → 짧다.

### 규칙

- **입력값 ≤ 범위 최소** → 수치 생략(어차피 전부 통과). 예) 희귀도(8-12) 8이상 = `템.희`
- **입력값 > 범위 최소** → 그 값 ~ 범위 최대만.
  - 희귀도(8-12) 11이상 = `템.희.*1[12]%`
  - 경로석수량(30-40) 35이상 = `견.*경.*(3[5-9]|40)%`
  - 성소확률(70-100) 90이상 = `성.가.*(9.|100)%`
- **범위 상한 없는 옵션 6종** (희귀도/무리/효율 등) → 입력값~999.
  - 아이템 희귀도 15이상 = `아이템.*(1[5-9]|[2-9].|\d..)%`
- **부활 횟수** 0~6, % 없음. 3이상 = `부활.*[3-6]`
- **단위**는 원문 따라 %/개/마리/초. 유적(1-2)개 2이상 = `탐.의.*2개`

### 범위 정규식 기법 (poe2.re 동일)

- `.` 자리채움: `[0-9]` 자리를 `.`로. 40~99 = `[4-9].`
- 첫자리 병합: `[2-8].`+`9.` → `[2-9].`
- 3자리 전체: `\d..` (게임 값에 앞자리 0 "099%" 없어 안전, `[1-9]..`보다 짧음)
- 압축: `[8-9]`→`[89]`, `[9-9]`/`[5-5]`→단일숫자
- 그룹은 `(...)` (게임 검색은 캡처 무의미 → `(?:` 불필요, 2글자 절약)
- 예) 33이상 = `(3[3-9]|[4-9].|\d..)%`

## 4. 조각 조합

- 포함 OR: `"조각1|조각2"`
- 포함 AND: `"조각1" "조각2"` (각각 세트)
- 제외: `"!조각"`
- 포함+수치: `"조각.*(범위식)단위"`

예) 탐험 유적 2개 이상 그리고 아이템 수량 15 이상:
```
"탐.의.*2개" "아.*수.*(1[5-9]|20)%"
```

### 유사어 배제 (?!...) — 필요 옵션에만 수동

영문 poe2.re의 `rar..(?!ch)`처럼, 유사어를 배제할 때 전방 부정. 우리는 고유성 조각이라 대부분 불필요하나, 특정 옵션이 겹치면 개별 적용.

## 5. 불리 옵션 숨기기 (경로석 접미)

```
!쇠약화|사슬|원.*약|점화|얼음|감전|재사용|충전량|생.*회
```

(쇠약화·시간의 사슬·원소 약화 저주, 점화/얼음/감전 지대, 재사용 대기시간 감폭, 플라스크 충전량 감소, 생명력·에너지 회복 감폭)

---

## 6. 거래소(공식 trade) 검색

- 시트 `거래소명(#)` = 정식 스탯명(#=수치), `stat id` = API용 ID.
- 전체 스탯 API: `GET https://poe.kakaogames.com/api/trade2/data/stats` (같은 도메인)
  - 글로벌: `https://www.pathofexile.com/api/trade2/data/stats`
- 경로석 옵션·접두는 거래소 검색 대상 아님(비움). 서판 3건(환영 안개 느리게, 공물 감소 2건)은 반대 부호라 실측 필요.

---

## 7. 검증 원칙

- 옵션 원문은 poe2db.tw/kr에서 확인(범위 표기 그대로).
- 조각은 생성 후 실매칭 검사: 그 옵션에 매칭 + 같은 풀 고유성(또는 중첩 수) 확인.
- 수치식은 경계값 전수 검증(예: 33~999, 80~120 등 자릿수 넘나듦 포함).
- 실전 테스트로 반대부호 3건·경로석 거래소명·중첩 심한 조각 보정.
